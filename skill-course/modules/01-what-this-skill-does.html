The four phases, named

Every gallery follows the same blueprint, written down in the skill’s instructions. Here are the phases by their real names — the same words the skill uses internally.

1
Codebase Analysis

Read files, trace flows, find the “cast” — who are the main characters in this project?

2
Curriculum Design

Decide on 4–10 modules and pick a simple or complex build path.

½
Phase 2.5 — Module Briefs

Write one brief per module — but only on the complex path. (We’ll meet The Brief later.)

3
Build the Course

Write the module HTML — one at a time, or many in parallel.

4
Assemble & Open

build.sh runs and produces index.html — the finished, openable course.

What comes out the other end

The skill doesn’t hand you one giant file. It produces a tidy directory — a labeled storage room where each item has its place.

Read the room layout on the left; the plain-English placards are on the right.

CODE 
course-name/
  styles.css       ← copied verbatim (never regenerated)
  main.js          ← copied verbatim (the interactivity engine)
  _base.html       ← shell: title, accent, nav dots
  modules/01-*.html … ← the only files written fresh
  index.html       ← assembled by build.sh
PLAIN ENGLISH

A folder named after the course — the whole gallery lives in here.

The stylesheet (colors, spacing, fonts) is copied as-is, never re-invented. One trusted look, every time.

The interactivity engine is also copied as-is — the quizzes and animations you’ve been clicking run on this one file.

A reusable shell — this is the boilerplate page frame that holds the title, accent color, and nav dots.

The module files — the only thing written fresh for your project. Everything unique lives here.

The finished page. build.sh assembles the shell + modules into this single, openable file.

🔗
Why “self-contained” matters

The course is self-contained: you double-click index.html and it opens like a document. No server to start, nothing to install — which is exactly why the styles and engine are copied rather than generated. A trusted engine, reused, beats a fresh one that might break.
