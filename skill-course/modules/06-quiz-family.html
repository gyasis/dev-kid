The Quiz Family

How a 10-line quiz grades itself — and the giant idea hiding inside it

Picture a game-show buzzer panel

You lock in a choice and slam the buzzer. The panel instantly lights green or red and shows you why — no host, no waiting, no calling a judge backstage.

Here's the secret: the answer key was wired into the panel before the show started. The "grading" is just the panel checking your lit-up button against the answer it already knew.

🔔
The reveal

Every quiz in this course knows its own answer and its own explanations — before you ever click. The grading is one line of code comparing two pieces of text.

The answer key lives in the panel itself

In our code, the panel is a chunk of HTML. The right answer and both explanations are baked into it as attributes.

CODE 
<div class="quiz-question-block"
     data-correct="option-b"
     data-explanation-right="Exactly — because X is responsible for Y."
     data-explanation-wrong="Not quite. Think about where Y lives...">
  <button class="quiz-option" data-value="option-b" onclick="selectOption(this)">
    <div class="quiz-option-radio"></div><span>Answer B</span>
  </button>
  <div class="quiz-feedback"></div>
</div>

Notice what's NOT here

There is no grading logic in this panel. No "if/then." It only describes the truth: here's the answer, here's what to say. The thinking happens elsewhere.

The entire "grader" is one comparison

When you press the check button, one shared piece of function runs. It reads the button you lit, compares it to the answer key, and reveals the matching explanation. That's the whole machine.

CODE 
if (selected.dataset.value === correct) {
  selected.classList.add('correct');
  feedback.innerHTML = '<strong>Exactly!</strong> ' + rightExp;
} else {
  selected.classList.add('incorrect');
  const correctBtn = $(`.quiz-option[data-value="${correct}"]`, q);
  if (correctBtn) correctBtn.classList.add('correct');
  feedback.innerHTML = '<strong>Not quite.</strong> ' + wrongExp;
}
PLAIN ENGLISH

IF the label of the button you pressed exactly matches the answer key…

…paint your button green ("correct").

…and show the author's pre-written praise.

Otherwise (you missed)…

…paint your button red ("incorrect").

…go find the button that was the answer…

…and light it green so you can see the right choice.

…then show the author's gentle nudge.

Done — that's the entire grading brain.

Same family, different panels

The big idea: describe the WHAT, not the HOW

This style has a name engineers use constantly: declarative. The data declares the truth; one generic engine reads that truth and acts on it.

Aha! — why this matters to you

Declarative design is one of the biggest ideas in modern software. Once you spot it, you'll see it everywhere — and you can tell an AI agent "keep this declarative" to get clean, reusable code.
