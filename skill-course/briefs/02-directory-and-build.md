# Module 2: The Directory & build.sh (low-level)

### Teaching Arc
- **Metaphor:** **Flat-pack furniture.** The hard parts (panels, screws) come pre-made in the box (`styles.css`, `main.js`); the instruction sheet just bolts them together (`build.sh`). You never re-mill the wood — you only arrange the panels. (No restaurant.)
- **Opening hook:** "A course is a folder, not a file. That choice is the secret to why these courses are consistent AND cheap to generate."
- **Key insight:** CSS and JS are **copied verbatim** (never regenerated by the AI), each module is a standalone `<section>` fragment, and `build.sh` is a one-line `cat` that glues `_base.html` + every `modules/*.html` + `_footer.html` into `index.html`.
- **"Why should I care?":** This is a reusable architecture pattern — *fixed engine, swappable content.* Recognizing it helps you design AI pipelines that don't waste tokens re-writing boilerplate every run.

### Code Snippets (pre-extracted)

File: build.sh (the entire assembler — yes, really)
```bash
#!/bin/bash
set -e
cat _base.html modules/*.html _footer.html > index.html
echo "Built index.html — open it in your browser."
```

File: _base.html (the shell — links the verbatim engine, opens <main>)
```html
<link rel="stylesheet" href="styles.css">
<style> :root { --color-accent: #2A7B9B; ... } </style>
<script src="main.js" defer></script>
...
<main id="main">
<!-- modules/*.html content is assembled here by build.sh -->
```

File: a module fragment (what an agent actually writes — no <html>/<head>/<body>)
```html
<section class="module" id="module-2">
  ... screens, visuals, quizzes ...
</section>
```
